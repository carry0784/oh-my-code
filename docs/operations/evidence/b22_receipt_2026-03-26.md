# B-22 헌법 조항 대조 검수본 — Scenario Viewer (설계 봉인)

---

## 공식 정의

> **B-22 Scenario Viewer는 기존 data/v2 값만 사용하여, 사전 정의된 목표 상태와 현재 상태의 조건 차이를 read-only로 비교 표시하는 Observation 패널이다.**

부제: State Comparison / Condition Comparison / Read-only comparison from existing values

---

## 1. 요구 조항

### A. 목적

| 항목 | 정의 |
|------|------|
| "시나리오"의 의미 | 미래 예측이나 추천이 **아니라**, 가정된 목표 상태를 기준으로 현재 상태를 비교 표시 |
| 패널의 역할 | 현재 시스템이 특정 기준 상태와 얼마나 일치/불일치하는지를 보여줌 |
| 판단 권한 | 사용자에게 실행 판단을 내리지 않음 |
| 데이터 처리 | 기존 값을 재배열/비교해 보여줄 뿐, 미래 상태를 추정하지 않음 |

### B. 허용 범위

| 허용 항목 |
|----------|
| 기존 `data/v2` 값 사용 |
| 현재값 vs 필요값 표시 |
| 충족/미충족 표시 |
| enum/bool/threshold/policy 조건 비교 |
| 총 충족 개수 집계 |
| 단순 비율 기반 진행률 표시 |
| read-only UI 렌더링 |
| 조건 비교 결과의 시각화 |

### C. 금지 범위

| 금지 항목 |
|----------|
| 추천 금지 |
| 예측 금지 |
| AI 판단 금지 |
| 신규 점수 생성 금지 |
| 신규 파생 지표 생성 금지 |
| 성공 가능성 추정 금지 |
| 승인 가능성 추정 금지 |
| next-step 자동 제안 금지 |
| 우선순위 재계산 금지 |
| 리스크 재해석 금지 |
| 미래 상태 시뮬레이션 금지 |

### D. Gap 계산 규칙

| 규칙 | 정의 |
|------|------|
| gap의 의미 | 예측 거리나 확률이 **아니다**. 조건 충족 여부를 보여주는 **비교 상태** |
| 상태형 (enum) | 일치/불일치 |
| 불리언형 (bool) | 충족/미충족 |
| 임계형 (threshold) | 기준 미달/충족 (Ops Score만 현재값/기준값 표기 허용) |
| 정책형 (policy) | 일치/불일치 |
| 새 계산 | 추가 금지. "충족 수 / 전체 수" 단순 집계만 허용 |

### E. Readiness Progress 의미 제한

> Readiness Progress는 전체 비교 조건 수 대비 현재 충족 조건 수의 단순 비율 표시이다.

| 제한 |
|------|
| 이 값은 **확률이 아니다** |
| 이 값은 **예측이 아니다** |
| 이 값은 **승인 가능성을 의미하지 않는다** |
| 이 값은 **거래 가능성을 의미하지 않는다** |
| 이 값은 단지 **조건 충족 개수의 비율 표시**이다 |

### F. B-19 / B-22 계층 역할 경계

| | B-19 Execution Pipeline | B-22 Scenario Viewer |
|---|---|---|
| 질문 | 현재 실행 흐름에서 어디가 막혔는가 | 사전 정의된 목표 상태 기준으로 무엇이 부족한가 |
| 관점 | pipeline blockage 관측 | condition comparison 관측 |
| 표시 | 6단계 순차 흐름도 + 차단 지점 | 7조건 현재값 vs 필요값 테이블 |
| 예측 기능 | **없음** | **없음** |

### G. 용어 오해 방지

- **명칭 유지**: "Scenario Viewer"는 카드 계보 내 위치 식별용으로 유지
- **부제 병기**: 내부 구현·검수에서는 "State Comparison" 또는 "Condition Comparison"으로 참조
- **오해 방지 이유**: "Scenario"가 시뮬레이션/예측으로 오해될 수 있으므로, 설계 문서와 검수본에서 "read-only comparison from existing values" 문구를 병기
