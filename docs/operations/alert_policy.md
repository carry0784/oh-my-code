# Alert Policy
## 알림 정책 v1.0

**Document Path:** `docs/operations/alert_policy.md`
**Parent Authority:** `Operating Constitution v1.0`
**Status:** ACTIVE
**Scope:** 알림 등급·필드·억제 규칙 세부화
**역참조:** 제14조~제22조

---

## 제약 선언

- 본 문서는 상위 헌법의 구현 세부화 문서이다.
- 본 문서는 새 권한을 추가하지 않는다.
- 본 문서는 자동 실행 권한을 부여하지 않는다.
- 본 문서는 상위 헌법과 충돌해서는 안 된다.
- 알림은 운영 개입 유도 장치이며, 자동 실행 권한을 가지지 않는다.

---

## 1. 목적 (제14조)

알림 시스템은 로그 전달 수단이 아니라 운영 개입 유도 장치다.
알림은 위험을 즉시 전달하고, 경보 피로를 억제하며, 조치 방향을 명확히 제시해야 한다.

---

## 2. 등급 체계 (제15조)

알림 등급은 다음 4단계로 제한한다. 이 목록 이외의 등급은 추가할 수 없다.

| 등급 | 용도 |
|------|------|
| `INFO` | 즉시 개입 불필요, 운영상 의미 있는 상태 |
| `WARNING` | 상태 악화 감지, 치명 차단 전 단계 |
| `CRITICAL` | 즉시 운영자 개입 필요한 치명 상태 |
| `PROMOTION` | 승격, 보류, 강등, 자본 확대·축소 판단 |

---

## 3. INFO (제16조)

즉시 개입이 필요하지 않으나 운영상 의미가 있는 상태를 알린다.

대표 사건:
- Daily Check OK
- Recovery complete
- 정상 상태 회복
- Observation 통과

---

## 4. WARNING (제17조)

상태 악화가 감지되었으나 치명 차단 전 단계인 경우에 발송한다.

대표 사건:
- Snapshot stale
- API poll 연속 실패
- Latency 악화
- 부분 mismatch
- Evidence 생성 지연

**운영자 해야 할 일 필드가 필수다.**

---

## 5. CRITICAL (제18조)

즉시 운영자 개입이 필요한 치명 상태에 발송한다.

대표 사건:
- LOCKDOWN 진입
- 거래 권한 차단
- 정합성 붕괴
- Ghost order 의심
- Kill switch 발동
- 무증거 거래 허용 시도

**운영자 해야 할 일 필드가 필수다.**

---

## 6. PROMOTION (제19조)

승격, 보류, 강등, 자본 확대·축소 판단에 발송한다.

**운영자 해야 할 일 필드가 필수다.**

---

## 7. 표준 필드 (제20조)

모든 알림은 다음 8개 필드를 포함해야 한다.

| 필드 | 설명 |
|------|------|
| 레벨 | INFO / WARNING / CRITICAL / PROMOTION |
| 시스템 상태 | 현재 시스템 상태 문구 |
| 사건명 | 발생 사건 식별 명칭 |
| 발생 시각 | 절대시간 (ISO 8601) |
| 영향 범위 | 영향받는 시스템/기능 |
| 자동 조치 결과 | 시스템이 취한 자동 조치 |
| 운영자 해야 할 일 | 운영자의 필수 조치 (WARNING 이상 필수) |
| Evidence/Receipt ID | 관련 증거/영수증 ID |

---

## 8. 알림 원칙 (제21조)

| 원칙 | 설명 |
|------|------|
| **Actionable First** | 조치 가능한 정보를 우선 전달한다 |
| **Noise Suppression** | 경보 피로를 억제한다 |
| **State Change Driven** | 상태 변화 시에만 발송한다 |
| **Recovery Must Notify** | 복구 알림은 생략할 수 없다 |
| **Evidence Attached** | 증거를 첨부한다 |

---

## 9. 억제 규칙 (제22조)

1. 동일 WARNING 반복은 묶음 처리한다.
2. 동일 CRITICAL 지속은 상태 변화 시에만 즉시 발송한다.
3. 복구 알림은 생략할 수 없다.
4. 단순 로그는 알림으로 승격할 수 없다.

---

## 10. 금지 사항

- 알림만으로 거래 승인 또는 상태 해제 허용 금지
- 알림만으로 자본 확대 또는 승격 실행 허용 금지
- 단순 로그를 알림으로 무분별하게 승격하는 정책 금지
- 상위 헌법에 없는 신규 알림 등급 추가 금지
- CRITICAL 복구 알림 생략 허용 금지
- 알림은 자동 실행 권한을 가지지 않는다.
